Overview

When a member redeems Cash Back as a gift card, we currently require a mailing address on file. Members without one hit a dead end in the redemption flow. This feature introduces an inline address collection step — surfaced as a modal between the gift card index and the order review page — so members can complete redemption without leaving the flow or navigating to Account Settings.

Problem

Members selecting a gift card are blocked from redeeming if no mailing address exists on their account. The current resolution path sends them to Account Settings, breaking the redemption flow and creating drop-off. First-time redeemers are disproportionately affected.

Goal
Primary goal Eliminate address-related drop-off in the gift card redemption funnel without requiring members to leave the flow.
User Story

As a member redeeming Cash Back as a gift card for the first time, I want to be prompted to enter my mailing address in context so I can complete my redemption without being redirected to another page.

Flow
1
Member lands on Gift Card index and selects a card
2
If no mailing address is on file — address modal appears over the dimmed index page
3
Member enters or confirms name and mailing address, taps Save & Continue
4
Member proceeds to Order Review and completes redemption
5
Address is saved to account — modal is never shown again for this member
If a mailing address already exists on the account, step 2 is skipped entirely.
Modal Design

The modal appears as an overlay on the gift card index. The background page is dimmed but remains visible to preserve context. It mirrors the existing address UI in Account Settings.

Pre-fills first/last name from account
Explains context without alarming the member
Fields: street, address 2, city, state, ZIP, country
X closes modal and returns to gift card index
CTA: Save & Continue → Order Review
Address persists to account on save
Framing copy "A mailing address is needed to send your gift card. Add it once and you're set for every redemption going forward."
Out of Scope
Address editing post-redemption (existing Account Settings flow handles this)
Showing the saved address on the Order Review page
International address formats beyond US / Canada
Success Metrics
Metric What we're measuring
Redemption drop-off rate Reduction in abandonment for members with no address on file
Modal completion rate Save & Continue vs. dismiss (X) — indicates friction level
Overall funnel abandonment No measurable increase vs. baseline for all redeemers
Dependencies
Address save API (same endpoint as Account Settings)
Account address lookup to gate modal display
Verification email trigger on address save
Session state to pass address into Order Review
